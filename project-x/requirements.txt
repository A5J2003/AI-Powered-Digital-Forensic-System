# KEY CHANGES vs original:
#
# 1. tensorflow 2.16.1 → 2.17.0
#    TF 2.16 introduced Keras 3 which broke tensorflow.python.trackable.
#    2.17.0 stabilises the Keras 3 migration and restores compatibility
#    with transformers and mtcnn on Python 3.12.
#
# 2. tf-keras KEPT (now pinned to ==2.17.0)
#    Required explicitly because TF 2.16+ ships Keras 3 as default.
#    tf-keras provides the legacy Keras 2 API that transformers,
#    mtcnn, and other packages still depend on internally.
#    Without it you get: No module named 'tensorflow.python.trackable'
#
# 3. transformers 4.46.3 → 4.44.2
#    4.46.x has a hard dependency on the Keras 3 internal API.
#    4.44.2 is the last release fully compatible with tf-keras shim
#    on TF 2.17 without requiring manual KERAS_BACKEND env var hacks.
#
# 4. torch 2.2.2 → 2.3.1 + matching torchvision/torchaudio
#    2.2.2 has a known Python 3.12 multiprocessing bug that causes
#    silent DataLoader hangs. 2.3.1 is the first fully stable 3.12
#    release. torchvision and torchaudio bumped to match exactly.
#
# 5. numpy 1.26.4 — KEPT as-is
#    This is the last numpy 1.x release. numpy 2.x breaks torch 2.3,
#    librosa, and opencv simultaneously. Do not upgrade to 2.x.
#
# INSTALL ORDER (important — run in this sequence):
#   pip install torch==2.3.1 torchvision==0.18.1 torchaudio==2.3.1
#   pip install tensorflow==2.17.0 tf-keras==2.17.0
#   pip install -r requirements.txt
# =====================================================================
